Class Collections

  • java.lang.Object
    • java.util.Collections

  • public class Collections
extends Object
    The Collection Framework in Java is a set of classes and interfaces that help developers manage collections, which are essentially groups of elements

    Historical Context and Drawbacks of Older Approaches

    Before Java version 1.2 (or Java 2), Java relied on specific classes like Vector, Stack, Hashtable, and Arrays to manipulate groups of objects or collections. However, these older classes suffered from several significant drawbacks: 1.Inconsistencies: Each class had its own style of managing data, leading to confusion and difficulty in remembering how to use them.2.Lack of Interoperability: The classes did not work well together. 3.Absence of Common Interfaces: There was no common interface, which made it challenging to write generic methods or define generic variables that could work across different collection types.

    Introduction of the Collection Framework

    The Collection Framework was introduced in Java version 2 (or 1.2) specifically to solve these problems. Its introduction brought several advantages: Interoperability: It enabled different collection types to work together seamlessly.Interchangeability: Elements and structures could be used interchangeably.Support for Generics: The presence of common interfaces allowed for the writing of generic methods and the definition of generic variables, making the code more flexible and reusable.

    Key Features of the Collection Framework

    • Collection: This is the root interface for almost all collection interfaces. It serves as a blueprint for basic operations such as checking size, emptiness, existence of elements, and adding or removing elements. Child interfaces like List, Set, and Queue extend the Collection interface. Collection itself cannot be directly instantiated as it's an interface.
    • Iterable: This is the root interface of the entire Collection Hierarchy. Any class that implements Iterable allows its objects to be iterated over using a for-each loop. The Collection interface extends Iterable, meaning all classes implementing Collection also support for-each loops
    • List: This interface represents ordered collections where elements are stored in the order they are added, similar to arrays. Lists allow duplicate elements. Key features include order preservation, index-based access, and allowing duplicates. Examples of classes implementing List include ArrayList, LinkedList, Vector, and Stack.
    • Set>: This interface does not allow duplicate elements. Elements in a Set are generally unordered.
    • Queue (Deque): This interface typically follows the First-In, First-Out (FIFO) principle, like a line at a doctor's office or a milk counter, where the first element added is the first one to be removed. Deque (pronounced "Deck"), or Double-Ended Queue, is a sub-interface under Queue that allows insertion and removal from both ends.
    • Map: Unlike other interfaces, Map is a separate interface and does not extend the Collection interface. It stores data as key-value pairs, similar to a dictionary. Keys in a Map must be unique, though values can be duplicated. An example is a student's roll number (key) mapped to their name (value).
